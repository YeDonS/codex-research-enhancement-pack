# Prompt 与 Goal 模板

## 1. 持续目标模板

```text
/goal
目标：围绕 <研究方向/项目> 建立可复用科研工作流，持续把输入材料转化为 evidence cards、method cards、研究问题、实验计划、handoff 和 review。

完成标准：
1. 每批输入先提炼可迁移方法和风险限制，不做摘要堆积。
2. 每个阶段至少产出一个可复用产物：skill、workflow、checklist、rubric 或模板。
3. 每个结论有来源定位或明确标注不确定。
4. 每轮结束更新 progress.md，包含输入材料、提炼结论、转化产物、验证结果、下一步。
5. 阶段结束做 review，删除泛泛建议，保留经验证的能力。
```

## 2. 小红书笔记批处理模板

```text
使用 xhs-method-ingester 处理以下笔记和评论：<链接/截图/正文>。
要求：
1. 先说明哪些材料可读，哪些不可读。
2. 提炼可迁移方法、操作步骤、适用场景、风险限制。
3. 只把具体、可验证的方法转成产物。
4. 至少输出 skill 草案、科研工作流模板、验收清单。
5. 更新 progress.md 的五栏记录。
```

## 3. 单篇论文精读模板

```text
使用 literature-evidence-reader 精读这篇论文：<PDF/DOI/路径>。
我的研究问题是：<问题>。
请先做 preflight，确认全文、图表、表格和 supplement 是否可读。
如果材料不足，先停止并列出缺口。
如果材料足够，输出 Paper Reading Pack：证据卡、方法卡、变量表、关键局限、可复现要点、Obsidian 入库候选笔记。
```

## 4. 文献综述模板

```text
围绕 <主题> 做文献调研。
范围：<年份/领域/方法/数据类型>。
排除：<不相关方向>。
输出：
1. 检索关键词和筛选规则。
2. 20 篇候选筛选表。
3. 3-5 篇精读候选和理由。
4. 证据矩阵。
5. related work 骨架。
不要只给论文列表。
```

## 5. 研究问题委员会模板

```text
使用 research-question-council 审查这个研究方向：<方向>。
输入材料：<论文/笔记/数据/约束>。
请用五个角色评审：反对者、第一性原理者、扩展者、局外人、执行者。
输出 3 个候选研究问题，并按新颖性、证据基础、可证伪性、可行性、贡献打分。
最后给出继续、缩小、转向或停止的裁决。
```

## 6. 实验设计模板

```text
为这个假设设计最小可运行实验：<假设>。
已知数据/代码：<路径或链接>。
限制：<时间/算力/样本/伦理>。
输出实验计划，包括变量、基线、指标、统计方法、运行命令、输出路径、失败标准和下一轮决策规则。
先设计最小实验，不要直接全量跑。
```

## 7. 长期任务 Handoff 模板

```text
使用 research-handoff-review 结束本轮任务。
请写 handoff，包含目标、范围、改动文件、命令/分析、产物路径、未完成事项、风险、下一步。
然后用独立 review 视角检查本轮产物，只返回问题清单和修正建议。
```

## 8. 粗略研究目标编译模板

```text
把下面一句话需求编译成可执行研究任务，不要写 prompt 教程。

原始目标：<一句话目标>
现有上下文：<项目/材料/已有产物>

请输出：
1. Goal Brief：outcome、范围、明确不做什么、交付物、完成证据、约束和合理假设。
2. 只在会改变安全、外部状态或可验证性时提 1-3 个关键问题；其余信息用假设继续。
3. 若任务涉及检查、debug、重构或修复，先给 inspection plan；检查完成后再给修复方案和影响，等待批准后修改。
4. 一条当前可执行的 next action。
```

## 9. 项目对话压缩模板

```text
使用 research-knowledge-curator 和 research-program-manager 整理这些项目对话。

输入：
- 会话导出或 thread IDs：<...>
- 项目目录：<...>
- 关键日志/代码/实验结果：<...>
- 当前目标：<...>

要求：
1. 先建会话清单和 source audit，不直接复述聊天。
2. 把每个重要说法标成 verified fact、model assumption、unverified claim、user decision 或 open question。
3. 聚合重复问答，只保留 canonical answer、证据位置和下一项诊断。
4. 前后矛盾的解释保留为 decision，不允许用最后一条回答静默覆盖。
5. 输出 issue register、decision/assumption log、do-not-repeat 和不超过一页的 context capsule。
6. 私有对话与本机路径只写入项目私有 handoff，不进入公开 repo。
```
