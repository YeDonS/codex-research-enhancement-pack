# 阶段 Review：2026-06-21 v0.5

## 本轮新增能力

- 新增包级验证器 `scripts/validate_pack.py`，把必需产物、skill 核心章节、进度五栏、UI 元数据、eval fixture 和 eval output 纳入结构守门。
- 新增 4 个独立 eval 输出，避免只在验证总结里保留不可复查的摘要。
- 修正 `literature-evidence-reader` smoke eval 的缺口：完整输出现在显式包含 Method Cards。

## 真正提升 Codex 能力的部分

- Eval output 文件让后续独立会话可以对照同一输入和同一输出格式，比较 skill 是否退化。
- 包级验证器降低了新增批次时漏掉 progress、review、rubric、fixture 或 skill 元数据的风险。
- 文献精读 eval 强制 Method Cards，使“论文拆解”从摘要/证据扩展到可复现方法层。

## 仍然只是结构验证的部分

- `validate_pack.py` 只证明文件和关键章节存在，不证明研究结论正确。
- Smoke eval 是同会话产物，不能证明跨会话稳定性。
- Fixture 是短文本，不覆盖真实 PDF 解析、图表识别、supplement 下载、Zotero/Obsidian 集成。

## 需要删除或重写的倾向

- 不应把 `PACK VALIDATION PASSED` 写成“科研能力已验证”。它只是包结构和最小样例输出守门。
- 不应把 fixture 输出当作真实研究结论引用。它们只用于验证 skill 输出形态。
- 若后续 eval 输出没有暴露缺点，也要主动加入 adversarial fixture，否则评估会变成形式化打勾。

## 下一步

1. 用独立会话重跑 4 个 fixture，并把结果和 v0.5 输出做差异审查。
2. 用真实 PDF 跑 `literature-evidence-reader`，优先验证图表、supplement 和页码定位。
3. 新增一个 adversarial fixture：材料中包含互相矛盾的论文结论，测试 evidence matrix 是否保留冲突而不是强行合并。
