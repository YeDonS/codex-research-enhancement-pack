- 更新 `reproduction-data-analyst`，把数据分析有效性审查从“数据版本/split/seed”扩展到 missingness、exclusion、label leakage、group leakage 和 sensitivity analysis。
- 更新 workflow、checklist、eval plan、README 和 validation 记录，让数据分析核验进入正式增强包。

## 真正提升 Codex 科研能力的部分

- Codex 不再只看显著性或 AUC，而会先检查分析管线是否能支持这些结论。
- 效应估计和预测模型被分开审查，避免把一个任务的通过标准误用于另一个任务。
- 下一步修正是最小诊断动作：missingness audit、grouped split、敏感性分析，而不是直接润色结论。

## 仍需警惕的部分

- Fixture 是合成输入，没有真实 notebook 执行、真实数据 schema 或统计模型代码。
- 当前输出能识别风险，但还不能自动选择最合适的因果估计方法。
- 对医学、金融、临床等高风险数据，仍需要领域专家确认缺失机制、混杂因素和伦理边界。

## 删除或重写标准

- 如果输出把 p = 0.018 或 AUC = 0.82 当成充分证据，应判失败。
- 如果只说“存在偏差风险”但不指出 treatment/severity missingness 和 user leakage，应重写。
- 如果建议扩大样本或继续写作，却没有先做 sensitivity analysis 和 grouped split，应重写。
