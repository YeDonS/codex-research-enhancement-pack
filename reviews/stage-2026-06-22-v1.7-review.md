# 阶段 Review：2026-06-22 v1.7

## 本轮输入

- 一个真实存储模拟器项目的代码、测试、SQLite/FIO 脚本和对话审计。
- 已安装的长期任务管理、复现分析和实验设计 skill。

## 真正提升 Codex 能力的部分

- 不可变 run manifest 将“看起来同配置”变成可机器拒绝的比较契约。
- 原始直方图与 sample conservation 能发现摘要分位数无法暴露的统计错误。
- mechanism counter gate 阻止把零触发路径写成性能变化原因。
- 最小合成 trace 能在不跑完整实验前暴露模拟器与物理设备语义差异。
- 静态测试、dry-run、目标内核运行必须分层验收，不能互相替代。

## 泛泛建议或应删除的部分

- “多记录日志”过于泛泛；必须明确记录 hash、flags、seed、workload、drain 和 loaded artifact。
- “比较 p99”不足以支持尾延迟结论；没有原始分布和样本守恒时应重写。
- “开启了功能所以功能生效”应删除；必须有 trigger/use counter 或 trace。
- “模拟器能重排所以硬件能抢占”应删除，除非有设备证据。
