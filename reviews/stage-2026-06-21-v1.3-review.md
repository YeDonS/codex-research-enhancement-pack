# 阶段 Review：2026-06-21 v1.3

## 本轮新增能力

- 新增 `literature-landscape-researcher`，覆盖可复现检索协议、query log、筛选、去重、证据图谱和 gap 审计。
- 新增 `literature-search-bias-minimal` adversarial fixture，拒绝 confirmation-only 搜索、单一数据库偏差和无依据 gap claim。
- 新增中文使用手册、安装/GitHub 同步说明和安全安装脚本。

## 真正提升 Codex 科研能力的部分

- 文献调研和单篇精读分层：先可复现检索/筛选，再全文证据提取。
- 检索过程保留 exact query、日期、数据库、过滤器、命中数、筛选理由和 dedupe 记录。
- Gap claim 必须与检索覆盖范围绑定，不能把“一个查询没搜到”写成“没有研究”。
- 安装脚本支持 list、dry-run、选择性安装和冲突保护，降低手工复制错误。

## 仍需警惕的部分

- Fixture 没有真实调用数据库 API，也没有验证不同数据库语法。
- 系统综述仍需要领域协议、双人筛选或正式 PRISMA 流程时的额外规则。
- GitHub 发布依赖本机完成 `gh auth login`，当前未完成远端同步。

## 删除或重写标准

- 如果输出迎合“只找支持证据”，应判失败。
- 如果没有 query/screening/dedupe log 就声称可复现，应重写。
- 如果一个窄查询未命中就声称 research gap，应重写。
- 如果安装脚本静默覆盖已有 skill，应判失败。
