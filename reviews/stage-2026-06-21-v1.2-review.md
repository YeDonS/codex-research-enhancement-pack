# 阶段 Review：2026-06-21 v1.2

## 本轮新增能力

- 新增 `research-knowledge-curator` skill，覆盖 Obsidian/Zotero/论文卡/证据卡/实验卡/索引笔记的整理和维护。
- 新增 `knowledge-curation-minimal` fixture 和输出，验证是否能避免摘要堆积、识别 abstract-only 弱证据、合并重复论文卡并保留冲突。
- 把知识库整理纳入 README、workflow、checklist、rubric、examples、eval plan、validation 和 progress。

## 真正提升 Codex 科研能力的部分

- 知识库写入前必须做 source audit 和 note plan，减少自动化制造垃圾笔记。
- 证据卡、论文卡、实验卡和索引笔记分层，避免把摘要、主张、实验日志混成一篇长笔记。
- 弱证据、重复项、冲突项和待补全文被显式记录，下一轮能接续。

## 仍需警惕的部分

- Fixture 没有真实 Obsidian vault，也没有实际写文件后的链接完整性检查。
- 不同用户的 vault 命名、frontmatter 和标签体系差异很大，真实使用时应优先遵守已有约定。
- 当前 skill 适合小批量整理；大规模迁移还需要脚本和 dry run。

## 删除或重写标准

- 如果输出只是把材料总结成长文，应判失败。
- 如果 abstract-only 或评论材料被写成强证据，应重写。
- 如果重复 DOI 产生两个无链接论文卡，应重写。
- 如果没有 write queue 和 review queue，下一轮无法维护，应重写。

## 下一步

1. 用真实 Obsidian/Zotero 样例验证路径、frontmatter、backlinks 和索引维护。
2. 增加一个 vault-consistency fixture，检查断链、重复 DOI、孤立证据卡和 stale claim。
3. 独立会话重跑 11 个 fixture，把失败项转成 skill 修订。
